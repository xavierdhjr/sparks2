Summary

Stores all preconfigured data that describe specific entities in the game.

Goals

  • Give client code access to information about the game.
  • Read data in from a data source.
    • A persistent data source, for development and the server.
    • A way to read data from a server.
  • Be configurable, so switching data sources is easy.

Data Context

Loading all information about the game at once is not a reasonable request. Instead, what can be done is the creation of a Data Context.

GameDataContext provides access to all of the Game's data by loading things on demand. If a script require's information for one item, there's no reason to load a hundred items.

How To Use

Loading Farm Item Info

Loading information about a farm item is a common task. If you have access to a GridObject, getting it's item name is simple.

// Create a context for the data
GameDataContext context = new GameDataContext();
GridObject gridObject = Obj; // public var on MonoBehaviour
FarmItemInfo itemInfo = context.GetItemInfo(gridObject.FarmItemId);
Debug.Log("This item is a " + itemInfo.ItemName);

Loading GridObjects

It's often necessary to create a GridObject in the scene. To do this, you need to get the GridObjectInfo that describes the object you want to load in.

// Create a context for the data
GameDataContext context = new GameDataContext();
// Go through each FarmItemInfo in the database and check its name
for (int i = 0; i < context.FarmItems.Length; ++i)
{
FarmItemInfo farmItemInfo = context.FarmItems[i];
if (farmItemInfo.ItemName == "Field of Wheat")
{
GridObjectInfo objInfo = context.GetObjectInfo(farmItemInfo.ItemId);
if (objInfo == null)
{
Debug.LogError("Could not find Field of Wheat GridObject");
break;
}
// Create a new GridObject based on this info
GridObject obj = objInfo.LoadObject(context);
Debug.Log(obj.FarmItemId);
}
}
