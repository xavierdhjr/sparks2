Summary

This system manages the creation, hiding, and showing of highlights for GridObjects. Highlights are outlines that show the area a GridObject occupies on the ground, and can be colored depending on the situation.

Goals

  • Show the area a GridObject occupies.
  • Show the area in different ways depending on what other objects occupy the area or what the GridObject is currently doing.
  • Be able to control the highlight of possibly many GridObjects.

Data Format

How to enable highlighting on a GridObject

You need to add the <capability>highlightable</capability> tag to a GridObject's data to enable highlighting.

<gridobject item-id="farm_item_1a" width="10" height="10">
<art-asset>GridObjects/farm_item_1a</art-asset>
<art-asset>GridObjects/farm_item_1b</art-asset>
<art-asset>GridObjects/farm_item_1c</art-asset>
<capability>draggable</capability>
<capability>highlightable</capability>
<capability>cropHarvestable</capability>
<layer>plots</layer>
</gridobject>

How To Use

Highlighting is done by setting HighlightState information and then applying that state to a given GridObject.

It's possible to set information once and not change it, or to set and clear it every frame.

void Update ()
{
if (Input.GetMouseButtonDown(0))
{
_clicks++;
}
// Reset all existing highlights
GridHighlightSystem.Get.ClearAll();
// Configure what kind of highlight based on
// number of clicks.
switch (_clicks)
{
case 0:
GridHighlightSystem.Get.ChangeHighlight(GridHighlightSystem.HighlightState.CanUse);
break;
case 1:
GridHighlightSystem.Get.ChangeHighlight(GridHighlightSystem.HighlightState.CannotUse);
break;
case 2:
GridHighlightSystem.Get.ChangeHighlight(GridHighlightSystem.HighlightState.ShowSpaceTaken);
break;
default:
_clicks = 0;
break;
}
// Actually change the highlight for this object
GridHighlightSystem.Get.SetHighlightState(Obj);
}
